----------------------------------------------------------------
SLIDE 1 — THE PROBLEM
----------------------------------------------------------------

Title: "Practising interviews out loud is hard and expensive."

  Key Points:
  - 75% of candidates say lack of realistic practice is their
    #1 reason for underperforming in interviews
    (LinkedIn Talent Trends, 2023)

  - Career coaches cost $100–300 per session — out of reach
    for most junior and mid-level candidates

  - Existing tools (ChatGPT, Glassdoor, LeetCode) are text-based
    and generic — not voice, not tailored to your specific role

  The Gap:
    No tool gives you a real conversational, voice-first interview
    experience for your specific job description, at near-zero cost.

----------------------------------------------------------------
SLIDE 2 — THE SOLUTION
----------------------------------------------------------------

Title: "MockMate — Your AI voice interviewer, 24/7, for ~$0.002."

  How it works:
    1. Paste any job description
       -> Get 5 tailored questions in seconds

    2. AI interviewer "Alex" speaks to you out loud
       -> You answer by voice (no typing)

    3. Receive a fully scored debrief
       -> Per-answer feedback, tips, ideal answers

    4. Download a PDF report
       -> Review it before the real interview

  Optional: Upload your CV to personalise every question to
            your actual experience and past companies.

----------------------------------------------------------------
SLIDE 3 — TECH STACK & WHY
----------------------------------------------------------------

Title: "Every technical choice was made to keep cost at zero
        and latency at real-time."

  +------------------+-------------------+----------------------+
  | Layer            | Choice            | Why                  |
  +------------------+-------------------+----------------------+
  | LLM              | Claude Haiku      | Fastest + cheapest   |
  |                  | (claude-haiku-4-5)| Anthropic model.     |
  |                  |                   | ~$0.002/session.     |
  |                  |                   | Real-time speed.     |
  +------------------+-------------------+----------------------+
  | Speech-to-Text   | Web Speech API    | FREE. Built into     |
  | (STT)            | (browser)         | Chrome. No Whisper,  |
  |                  |                   | no Deepgram cost.    |
  +------------------+-------------------+----------------------+
  | Text-to-Speech   | SpeechSynthesis   | FREE. Built into     |
  | (TTS)            | API (browser)     | all browsers.        |
  |                  |                   | No ElevenLabs cost.  |
  +------------------+-------------------+----------------------+
  | Backend          | FastAPI (Python)  | Async, fast to       |
  |                  |                   | prototype, clean     |
  |                  |                   | OpenAI-compatible    |
  |                  |                   | client.              |
  +------------------+-------------------+----------------------+
  | Frontend         | React + Vite      | Fast dev cycle,      |
  |                  | + Tailwind CSS    | easy to iterate UI.  |
  +------------------+-------------------+----------------------+
  | Database         | None (in-memory)  | Single-session tool. |
  |                  |                   | No persistence       |
  |                  |                   | needed. Zero infra.  |
  +------------------+-------------------+----------------------+
  | Hosting          | Render + Vercel   | Both free tier.      |
  |                  |                   | Zero hosting cost    |
  |                  |                   | at MVP scale.        |
  +------------------+-------------------+----------------------+

  Privacy note:
    MockMate stores no user data. All session data is in-memory
    and discarded when the session ends. The only external
    processor is Anthropic, which does not use API inputs for
    model training. Designed with FADP principles in mind:
    data minimisation, storage limitation, purpose limitation.

SLIDE 4 — ARCHITECTURE DIAGRAM
----------------------------------------------------------------

Title: "Input -> AI/Logic -> Output"

  JD + CV + Settings (user input)
          |
          v
     React Frontend
          |
     POST /parse-jd
          |
          v
     FastAPI Backend
          |
     Prompt assembled (JD + CV + difficulty + type + company)
          |
          v
     Claude Haiku (OpenRouter)
          |
     Returns 5 tailored questions
          |
          v
     Browser TTS speaks Question 1 aloud
          |
     User holds SPACE -> speaks answer
          |
     Browser STT transcribes voice -> text
          |
     POST /respond
          |
     +----+----+
     |         |
  < 20 words  OK
     |         |
  Follow-up  Acknowledge + speak next question
     |         |
     +----+----+
          |
     [Repeat for Q2 -> Q3 -> Q4 -> Q5]
          |
          v
     POST /debrief (all 5 Q&A pairs)
          |
          v
     Claude Haiku scores + evaluates each answer
          |
          v
     React renders Debrief UI
          |
          v
     jsPDF -> downloadable PDF report

  Cost:  STT = $0  |  TTS = $0  |  Claude Haiku = ~$0.002
