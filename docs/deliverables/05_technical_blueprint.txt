================================================================
MOCKMATE — DELIVERABLE 5: TECHNICAL BLUEPRINT
================================================================

Flow: Input -> Process (AI / Logic) -> Output

----------------------------------------------------------------
INPUT
----------------------------------------------------------------
  - Job Description text (pasted by user)
  - CV / Resume file (PDF, DOCX, or TXT — optional)
  - Difficulty level (Junior / Mid / Senior)
  - Interview type (Full / Behavioral / Technical / Screening)
  - Company name (optional)

----------------------------------------------------------------
PROCESS
----------------------------------------------------------------

  [1] CV EXTRACTION (if file uploaded)
      User uploads file
          |
          v
      FastAPI /extract-cv endpoint
          |
          v
      pdfplumber (PDF) / python-docx (DOCX) / decode (TXT)
          |
          v
      Text trimmed to 800 words max
          |
          v
      Returns cv_text to frontend

  ----------------------------------------------------------

  [2] QUESTION GENERATION
      Frontend sends JD + CV + difficulty + type + company
          |
          v
      FastAPI /parse-jd endpoint
          |
          v
      Prompt assembled:
        - JD text
        - CV summary (if provided)
        - Difficulty context (Junior/Mid/Senior instructions)
        - Interview type context (Behavioral/Technical/etc.)
        - Company context (culture/values if company known)
          |
          v
      Claude Haiku (via OpenRouter API)
          |
          v
      Returns JSON: { role, candidate_name, questions[5] }
          |
          v
      FastAPI builds intro message: "Hi [name], I am Alex..."
          |
          v
      Frontend receives questions and intro message

  ----------------------------------------------------------

  [3] LIVE INTERVIEW LOOP (repeats for each of 5 questions)

      Browser SpeechSynthesis API (TTS)
          |  speaks question aloud — FREE, no API cost
          v
      User holds SPACE or button to answer
          |
          v
      Browser Web Speech API (STT)
          |  transcribes voice to text — FREE, Chrome built-in
          v
      Answer text sent to FastAPI /respond endpoint
          |
          v
      Answer word count check:
          |
          +-- Under 20 words? --> Claude generates follow-up
          |                       prompt ("Could you give me
          |                       a specific example?")
          |                       --> TTS speaks follow-up
          |                       --> user answers again
          |
          +-- 20+ words? ------> Claude generates brief
                                  acknowledgement (5 words max)
                                  + asks next question exactly
                                  as written
                                  --> TTS speaks response

      Last question answered:
          |
          v
      Claude thanks candidate, says feedback is being prepared

  ----------------------------------------------------------

  [4] DEBRIEF GENERATION
      All 5 Q&A pairs sent to FastAPI /debrief endpoint
          |
          v
      Claude Haiku evaluates each answer on 3 dimensions:
        - Relevance to question (1-10)
        - Use of specific examples (1-10)
        - Clarity and structure (1-10)
          |
          v
      Returns JSON:
        {
          answers: [
            {
              question, answer_summary, score,
              feedback, tip, ideal_answer
            }
            ... x5
          ],
          overall_score,
          summary,
          top_strength,
          top_improvement
        }
          |
          v
      React renders Debrief UI with scores and feedback
          |
          v
      jsPDF generates downloadable PDF report

----------------------------------------------------------------
OUTPUT
----------------------------------------------------------------
  - Scored debrief: per-answer score (1-10), feedback, tip,
    ideal answer example
  - Overall score (average of all answers)
  - 3-sentence overall summary
  - Top strength and top improvement area
  - Downloadable PDF report

----------------------------------------------------------------
COST BREAKDOWN
----------------------------------------------------------------
  STT (Speech-to-Text):   $0.00  — Browser Web Speech API
  TTS (Text-to-Speech):   $0.00  — Browser SpeechSynthesis API
  LLM (Claude Haiku):    ~$0.001–0.003 per full session
  Hosting (Render):       $0.00  — Free tier
  Hosting (Vercel):       $0.00  — Free tier
  Database:               $0.00  — None (in-memory only)
  ─────────────────────────────────────────────────────
  Total per session:     ~$0.001–0.003

----------------------------------------------------------------
TECH STACK SUMMARY
----------------------------------------------------------------
  Frontend:   React 18 + Tailwind CSS (Vite)
  Backend:    FastAPI (Python)
  LLM:        Claude Haiku (claude-haiku-4-5) via OpenRouter
  STT:        Web Speech API — browser built-in, free
  TTS:        SpeechSynthesis API — browser built-in, free
  State:      In-memory (no database, no persistent storage)
  Auth:       None
  Deploy:     Render (backend) + Vercel (frontend)
